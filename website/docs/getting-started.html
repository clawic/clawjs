First Steps

This guide takes you from zero to a working ClawJS project with the official CLI flow: scaffold with claw new, initialize the workspace, and then extend the repo with generators and integrations.

Prerequisites

RequirementNotes
Node.js20+ required.
Package manager`npm`, `pnpm`, or `yarn`.
Runtime adapterChoose the adapter that matches your runtime. Support levels differ by adapter.

Install ClawJS

npm install -g @clawjs/cli
claw --help

claw new app my-app
cd my-app
npm run claw:init

Use @clawjs/cli for the global claw command. Generated projects already include @clawjs/claw for application code.

Extend the project

claw generate skill support-triage
claw generate provider openai
claw add telegram
claw info --json

The root claw.project.json file created by the starter is what makes these commands stable across project types.

Create an instance

import { Claw } from "@clawjs/claw";

const claw = await Claw({
  runtime: {
    adapter: "zeroclaw",
  },
  workspace: {
    appId: "my-app",
    workspaceId: "main",
    agentId: "assistant",
    rootDir: "./workspace",
  },
});

The `runtime` block can also include adapter-specific location overrides such as `homeDir`, `configPath`, `workspacePath`, `authStorePath`, and `gateway.configPath`.

Probe the runtime

const status = await claw.runtime.status();

console.log(status.adapter);
console.log(status.version);
console.log(status.capabilityMap);

Important: `capabilityMap` is the source of truth for optional features. Use it before assuming scheduler, memory, skills, channels, sandbox, or plugins exist.

Initialize the workspace

await claw.workspace.init();

const validation = await claw.workspace.validate();
if (!validation.manifest) {
  throw new Error("Workspace manifest missing");
}

ClawJS always creates its own stable `.clawjs/` layout, and then seeds the runtime-facing files from the selected adapter contract.

Refresh compat and inspect intent plus observed state

await claw.compat.refresh();

const snapshot = await claw.workspace.inspect();
console.log(snapshot.compatSnapshotPath);
console.log(snapshot.intentPaths.models);
console.log(snapshot.observedPaths.models);
console.log(snapshot.schedulerStatePath);
console.log(snapshot.memoryStatePath);

Work with providers and models

const providerCatalog = await claw.providers.catalog();
const modelCatalog = await claw.models.catalog();

console.log(providerCatalog.providers);
console.log(modelCatalog.defaultModel);

Start a conversation

const session = await claw.conversations.createSession("Hello");

for await (const event of claw.conversations.streamAssistantReplyEvents({
  sessionId: session.sessionId,
  transport: "auto",
})) {
  if (event.type === "chunk") {
    process.stdout.write(event.chunk.delta);
  }
}

Optional adapter-specific subsystems

const schedulers = await claw.scheduler.list();
const memory = await claw.memory.list();
const skills = await claw.skills.list();
const channels = await claw.channels.list();

If the adapter does not support one of these subsystems, the runtime capability map and doctor/compat reports will show that explicitly.
