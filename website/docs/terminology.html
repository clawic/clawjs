Terminology

This page is the canonical naming reference for ClawJS. Use these terms consistently across code, docs, examples, and product copy.

Canonical terms

Runtime adapter

A runtime adapter is the integration boundary between ClawJS and a concrete runtime such as openclaw, zeroclaw, or picoclaw. It owns capability probing, install/setup/repair flows, workspace contracts, auth, models, and conversation transport behavior.

Workspace

A workspace is the isolated operational directory managed by ClawJS. It combines the stable .clawjs/ layer with the runtime-facing files declared by the selected runtime adapter.

Agent

An agent is the identity and operational behavior that runs inside a workspace. An adapter may expose one agent or many agents; ClawJS still uses agent as the product term in both cases.

Agent profile

An agent profile is a named variant or configuration of the same agent. Use it when identity stays the same but defaults, behavior, credentials, or operating mode vary.

Provider and model

A provider is the external inference or service vendor. A model is the specific model identifier selected through that provider or through an adapter-backed model surface. They are related but they are not synonyms.

Gateway

A gateway is a mediation layer between clients, agents, and providers. Use gateway only for network or transport mediation concerns such as routing, policy, retries, and shared provider boundaries.

Concept hierarchy

  1. A runtime adapter defines capabilities and runtime behavior.
  2. A workspace is initialized against that adapter.
  3. An agent operates within that workspace.
  4. An agent profile can vary that agent's configuration.
  5. provider and model selection sit in the inference layer exposed by the adapter.
  6. A gateway may sit in front of runtimes or agents, but it is a separate concept.

Do not use as synonyms

  • gateway and runtime adapter
  • workspace and agent
  • provider and model

Scaffold defaults

Some examples and starters set workspaceId and agentId to the same value for convenience. That is a scaffold default, not a product rule.
